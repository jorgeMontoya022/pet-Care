# PetCare — Sistema de Gestión Veterinaria

Aplicación web desarrollada en Java para la gestión administrativa y clínica de veterinarias pequeñas y medianas. Permite controlar el ciclo completo de atención del paciente, desde el registro del dueño y su mascota hasta el seguimiento de citas médicas e historias clínicas.

## Funcionalidades

- Autenticación de veterinarios por tarjeta profesional
- Gestión de dueños y sus mascotas de forma relacional
- Programación y seguimiento de citas médicas
- Historia clínica cronológica por paciente
- Generación de reportes PDF con datos del paciente y dueño
- Validación de citas duplicadas por veterinario
- Protección de rutas por sesión activa

## Esquema de base de datos
```sql
DUENOS          → idDueno, cedula, nombre, telefono, correo
MASCOTAS        → idMascota, nombre, especie, raza, edad, idDueno (FK)
VETERINARIOS    → idVeterinario, nombre, especialidad, tarjetaProfesional, contrasena
CITAS           → idCita, idMascota (FK), idVeterinario (FK), fechaHora, motivo, estado
HISTORIASCLINICAS → idHistorial, idMascota (FK), descripcion, medicacion, fecha
```

## Instalación y ejecución

### Prerrequisitos
- Java 21+
- Apache Tomcat 10+
- Oracle Database XE
- Maven 3.8+

### Pasos

**1. Clona el repositorio**
```bash
git clone https://github.com/jorgeMontoya022/pet-Care.git
cd pet-Care
```

**2. Crea las tablas en Oracle**
```sql
CREATE TABLE DUENOS (
    idDueno NUMBER GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
    cedula VARCHAR(20) UNIQUE NOT NULL,
    nombre VARCHAR(100) NOT NULL,
    telefono VARCHAR(20),
    correo VARCHAR(100)
);

CREATE TABLE MASCOTAS (
    idMascota NUMBER GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
    nombre VARCHAR(50) NOT NULL,
    especie VARCHAR(50),
    raza VARCHAR(50),
    edad NUMBER,
    idDueno NUMBER,
    CONSTRAINT fkDuenoMascota FOREIGN KEY(idDueno) REFERENCES DUENOS(idDueno)
);

CREATE TABLE VETERINARIOS (
    idVeterinario NUMBER GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
    nombre VARCHAR(100) NOT NULL,
    especialidad VARCHAR(50),
    tarjetaProfesional VARCHAR(50) UNIQUE,
    contrasena VARCHAR2(100)
);

CREATE TABLE CITAS (
    idCita NUMBER GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
    idMascota NUMBER,
    idVeterinario NUMBER,
    fechaHora TIMESTAMP,
    motivo VARCHAR(200),
    estado VARCHAR(20) DEFAULT 'PROGRAMADA',
    CONSTRAINT fkCitaMascota FOREIGN KEY(idMascota) REFERENCES MASCOTAS(idMascota),
    CONSTRAINT fkCitaVet FOREIGN KEY(idVeterinario) REFERENCES VETERINARIOS(idVeterinario)
);

CREATE TABLE HISTORIASCLINICAS (
    idHistorial NUMBER GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
    idMascota NUMBER NOT NULL,
    descripcion VARCHAR2(1000),
    medicacion VARCHAR2(500),
    fecha DATE DEFAULT SYSDATE,
    CONSTRAINT fkHistoriaMascota FOREIGN KEY(idMascota) REFERENCES MASCOTAS(idMascota)
);
```

**3. Inserta un veterinario inicial para el primer login**
```sql
INSERT INTO VETERINARIOS (nombre, especialidad, tarjetaProfesional, contrasena)
VALUES ('Administrador', 'General', 'TP-001', '1234');
COMMIT;
```

**4. Configura la conexión en `Conexion.java`**
```java
private static final String URL  = "jdbc:oracle:thin:@localhost:1521:xe";
private static final String USER = "tu_usuario";
private static final String PASS = "tu_contraseña";
```

**5. Compila el proyecto**
```bash
mvn clean package
```

**6. Despliega en Tomcat y accede a:**
```
http://localhost:8080/PetCare
```
